### 一、项目概况与双轨架构
本项目是一个面向航空气象席位的本地化 Web 应用，包含两个核心子系统：
1. **“事中”气象综合监控告警系统（监控端）**：
   - 定时轮询抓取进出港航班计划与最新的 METAR/TAF 报文。
   - 对 7 大核心气象要素进行深度解析，结合航班时刻和各机场独立配置的告警阈值，触发桌面级红黄绿弹窗预警。
2. **“事后”预报质量自动化评定系统（评估计算器，吴霄独立开发模块）**：
   - **双模式**：支持 TAF 机场预报（全天候）与手工席位预报的客观打分。
   - **AMD 智能溯源**：能够拆分并追踪 TAF 的原始报与多次修订报（AMD-1, AMD-2...），分别定级。
   - **复杂算法**：按列（项目）独立统计空报、漏报。总分计算公式：`总分 = 准确率*50% + 完美率*30% + 优秀率*20% - 空报率*10% - 漏报率*10%`。项目未参评时不计入总评。
   - **数据引擎**：采用本地微型 JSON 数据库存储历史评定。支持将复杂双行表头的统计结果导出为 Excel，且具备强大的**双向逆向同步算法**，能从手工修改过的 Excel 表格（容错合并单元格等残缺）中重新抓取数据并刷新回 JSON 底层库。

### 二、技术栈与运行环境
- **后端**：Python 3 + Flask + Waitress（生产级 WSGI 服务器）。
- **底层数据库**：SQLite（处理高频航班与监控） + 层级化 JSON（处理复杂的质量评定历史）。
- **前端**：原生 HTML/CSS/JavaScript (ES6+) 的单页应用 (SPA)。图表使用 ECharts 5，日期组件使用 Flatpickr。
- **打包方式**：使用 PyInstaller 打包为 `ForecastTool.exe`，`console=False`（无控制台模式）。内置 `taskkill` 清理同名旧进程，并有 `waitress` 的防崩溃静默处理。
- **桌面穿透存储**：后端使用 `winreg` 注册表读取技术，穿透 OneDrive 劫持，将 Excel 精准导出至真实的物理桌面 `SF预报评定导出` 文件夹。

### 三、极客化 UI 与交互特性
1. **智能滚轮**：在各种日历/数字输入框悬停滚动鼠标滑轮，可无缝极速增减时间，同时屏蔽原生 input 的污染。
2. **复制与锁定**：前端表格支持点击表头“一键复制无表头行”，支持强制固定特定列的渲染顺序（风速、能见度、云高等排前）。
3. **离线与权限管理**：系统包含扫码登录与离线登录。最高管理员（工号41060711，吴霄）拥有特殊设置权限及“AMD识别”全局开关。
